什么是Vue CLI？

Vue CLI（Command Line Interface）是Vue.js官方提供的项目脚手架工具，它为Vue.js应用程序提供了完整的构建设置，无需手动配置构建工具。Vue CLI抽象了webpack的复杂配置，让开发者可以专注于编写应用程序代码。

安装Vue CLI

首先安装Vue CLI：

npm install -g @vue/cli # 或者使用yarn yarn global add @vue/cli

创建新项目

使用Vue CLI创建新项目非常简单：

vue create my-project

执行此命令后，Vue CLI会提供一个交互式界面，让你选择预设配置或手动选择功能：

? Please pick a preset: Default ([Vue 2] babel, eslint) Default (Vue 3) ([Vue 3] babel, eslint) ❯ Manually select features # 手动选择功能

如果选择手动配置，会看到以下选项：

? Check the features needed for your project: ❯◉ Choose Vue version ◉ Babel ◯ TypeScript ◯ PWA Support ◉ Router ◉ Vuex ◉ CSS Pre-processors ◉ Linter / Formatter ◯ Unit Testing ◯ E2E Testing

Vue CLI图形界面

Vue CLI还提供了一个图形用户界面，可以通过以下命令启动：

vue ui

这将打开一个浏览器窗口，提供可视化的项目创建和管理界面。

项目结构

Vue CLI创建的项目具有标准的目录结构：

my-project/ ├── public/ │ ├── index.html # 主页面模板 │ └── favicon.ico # 网站图标 ├── src/ │ ├── assets/ # 静态资源 │ ├── components/ # 组件 │ ├── views/ # 页面组件 │ ├── router/ # 路由配置 │ ├── store/ # Vuex状态管理 │ ├── App.vue # 根组件 │ └── main.js # 应用入口 ├── tests/ # 测试文件 ├── node_modules/ # 依赖包 ├── package.json # 项目配置 ├── babel.config.js # Babel配置 ├── vue.config.js # Vue CLI配置（可选） └── README.md # 项目说明

配置文件

Vue CLI项目可以通过多种方式配置：

vue.config.js

在项目根目录创建[vue.config.js](file:///e:\FrontendTeach\VUE快速上手\advanced\custom-elements.html#L134-L134)文件来自定义配置：

module.exports = { // 基本路径 publicPath: process.env.NODE_ENV === 'production' ? '/production-path/' : '/', // 输出目录 outputDir: 'dist', // 静态资源目录 assetsDir: 'static', // 开发服务器配置 devServer: { port: 8080, proxy: { '/api': { target: 'http://localhost:3000', changeOrigin: true, pathRewrite: { '^/api': '' } } } }, // webpack配置 configureWebpack: { resolve: { alias: { '@': path.resolve(__dirname, 'src') } } }, // CSS相关配置 css: { // 启用CSS modules modules: false, // 是否提取CSS到单独文件 extract: process.env.NODE_ENV === 'production', // 是否开启source map sourceMap: false, // CSS预处理器配置 loaderOptions: { sass: { additionalData: `@import "@/styles/variables.scss";` } } } }

常用命令

Vue CLI项目提供了多种命令来管理项目：

# 启动开发服务器 npm run serve # 构建生产版本 npm run build # 运行测试 npm run test:unit # 检查和修复文件 npm run lint

插件系统

Vue CLI的强大之处在于其插件系统，可以轻松扩展功能：

# 安装插件 vue add router # 添加Vue Router vue add vuex # 添加Vuex vue add eslint # 添加ESLint vue add babel # 添加Babel

插件系统遵循约定优于配置的原则，为项目提供最佳实践。

插件开发

你也可以开发自己的Vue CLI插件：

# 创建插件项目 vue add @vue/cli-plugin-eslint

一个典型的插件结构如下：

my-cli-plugin/ ├── package.json ├── generator/ # 生成器 │ ├── index.js # 生成器主文件 │ └── template/ # 模板文件 └── prompts.js # 交互式提示

环境变量和模式

Vue CLI支持环境变量和模式配置：

# .env # 全局环境变量 # .env.local # 本地环境变量（被git忽略） # .env.[mode] # 指定模式的环境变量 # .env.[mode].local # 指定模式的本地环境变量

环境变量示例：

NODE_ENV=production VUE_APP_TITLE=My App VUE_APP_API_URL=https://api.example.com

在代码中使用环境变量：

<template> <div> <h1>{{ title }}</h1> </div> </template> <script> export default { data() { return { title: process.env.VUE_APP_TITLE } } } </script>
