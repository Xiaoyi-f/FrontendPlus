插槽(Slots)

插槽(Slots)是Vue.js中内容分发的重要机制，它允许我们在组件中插入自定义内容。通过插槽，我们可以创建更加灵活和可复用的组件，让组件可以包含不确定的内容。

插槽核心概念

  • 默认插槽：组件中最基本的插槽形式
  • 具名插槽：为插槽指定名称，允许多个插槽
  • 作用域插槽：将子组件的数据传递给父组件
  • 插槽Props：传递数据给插槽内容
  • 插槽默认内容：当没有提供插槽内容时的默认显示
  • v-slot指令：使用v-slot指令分配内容到插槽

默认插槽

默认插槽是最简单的插槽形式，允许父组件的内容插入到子组件中没有明确插槽的位置。

默认插槽示例
<div id="app">
  <alert-box>这是一个警告消息</alert-box>
  <alert-box><strong>带HTML的警告消息</strong></alert-box>
</div>

<script>
const { createApp } = Vue;

createApp({})
  .component('alert-box', {
    template: `
      <div class="alert-box">
        <span class="icon">⚠️</span>
        <slot></slot>
      </div>
    `
  })
  .mount('#app');
</script>

具名插槽

具名插槽允许我们为插槽指定名称，这样就可以在组件中定义多个插槽，并将内容分发到特定的插槽中。

插槽类型 语法 说明 使用场景
默认插槽 <slot></slot> 没有指定名称的插槽 组件中只需要一个插槽
具名插槽 <slot name="slotName"></slot> 带有指定名称的插槽 组件中需要多个插槽
作用域插槽 <slot :data="item"></slot> 向插槽传递数据的插槽 需要向父组件传递数据
具名插槽示例
<div id="app">
  <layout-component>
    <template v-slot:header>
      <h1>页面标题</h1>
    </template>

    <template v-slot:main>
      <p>这是页面的主要内容。</p>
      <p>可以包含任意内容。</p>
    </template>

    <template v-slot:footer>
      <p>© 2023 版权信息</p>
    </template>
  </layout-component>
</div>

<script>
const { createApp } = Vue;

createApp({})
  .component('layout-component', {
    template: `
      <div class="layout">
        <header class="header">
          <slot name="header"></slot>
        </header>
        <main class="main">
          <slot name="main"></slot>
        </main>
        <footer class="footer">
          <slot name="footer"></slot>
        </footer>
      </div>
    `
  })
  .mount('#app');
</script>

作用域插槽

作用域插槽允许子组件向父组件传递数据，使得父组件可以使用子组件的数据来渲染插槽内容。

作用域插槽示例
<div id="app">
  <user-list :users="users">
    <template v-slot:default="slotProps">
      <div class="user-item">
        <span>{{ slotProps.user.name }} - {{ slotProps.user.email }}</span>
        <button @click="editUser(slotProps.user.id)">编辑</button>
      </div>
    </template>
  </user-list>
</div>

<script>
const { createApp, ref } = Vue;

createApp({
    setup() {
      const users = ref([
        { id: 1, name: '张三', email: 'zhangsan@example.com' },
        { id: 2, name: '李四', email: 'lisi@example.com' },
        { id: 3, name: '王五', email: 'wangwu@example.com' }
      ]);

      const editUser = (userId) => {
        alert(`编辑用户 ${userId}`);
      };

      return {
        users,
        editUser
      };
    }
    components: {
      UserList: {
        props: ['users'],
        template: `
          <div class="user-list">
            <div v-for="user in users" :key="user.id">
              <slot :user="user"></slot>
            </div>
          </div>
        `
      }
    }
}).mount('#app');
</script>

插槽Props

插槽Props允许我们将数据从子组件传递到父组件的插槽内容中，使得父组件可以使用子组件的数据来渲染内容。

插槽Props示例
<div id="app">
  <todo-list :todos="todos">
    <template #item="{ todo, index, isComplete }">
      <div class="todo-item">
        <span :class="{ completed: isComplete }">{{ index + 1 }}. {{ todo.text }}</span>
        <button @click="toggleTodo(todo.id)">{{ isComplete ? '取消完成' : '完成' }}</button>
      </div>
    </template>
  </todo-list>
</div>

<script>
const { createApp, ref } = Vue;

createApp({
    setup() {
      const todos = ref([
        { id: 1, text: '学习Vue基础', completed: false },
        { id: 2, text: '掌握组件通信', completed: false },
        { id: 3, text: '实践项目开发', completed: true }
      ]);

      const toggleTodo = (id) => {
        const todo = todos.value.find(t => t.id === id);
        if (todo) {
          todo.completed = !todo.completed;
        }
      };

      return {
        todos,
        toggleTodo
      };
    }
    components: {
      TodoList: {
        props: ['todos'],
        template: `
          <div class="todo-list">
            <div v-for="(todo, index) in todos" :key="todo.id">
              <slot
                name="item"
                :todo="todo"
                :index="index"
                :isComplete="todo.completed">
            </slot>
            </div>
          </div>
        `
      }
    }
}).mount('#app');
</script>

插槽默认内容

我们可以为插槽指定默认内容，当父组件没有提供插槽内容时，将显示默认内容。

插槽默认内容示例
<div id="app">
  <button-component>自定义按钮文本</button-component>
  <button-component></button-component>
</div>

<script>
const { createApp } = Vue;

createApp({})
  .component('button-component', {
    template: `
      <button class="btn">
        <slot>默认按钮</slot>
      </button>
    `
  })
  .mount('#app');
</script>

<style>

  background-color: #42b983;
  color: white;
  border: none;

  cursor: pointer;
}
</style>

插槽与组合式API

在组合式API中，插槽的使用方式与选项式API基本相同，但可以结合响应式数据进行更灵活的处理。

选项式API
const ListComponent = {
  props: ['items'],
  template: `
    <ul>
      <li v-for="item in items" :key="item.id">
        <slot :item="item">{{ item.name }}</slot>
      </li>
    </ul>
  `
};
组合式API
import { computed } from 'vue';

const ListComponent = {
  props: ['items'],
  setup(props) {
    const activeItems = computed(() => {
      return props.items.filter(item => item.active);
    });

    return {
      activeItems
    };
  },
  template: `
    <ul>
      <li v-for="item in activeItems" :key="item.id">
        <slot :item="item">{{ item.name }}</slot>
      </li>
    </ul>
  `
};

实际应用场景

插槽在实际开发中有很多应用场景：

常见应用场景

  • 布局组件：创建可定制的页面布局结构
  • 卡片组件：允许自定义卡片内容
  • 列表组件：允许自定义列表项内容
  • 模态框组件：允许自定义模态框内容
  • 按钮组件：允许自定义按钮内容和样式
  • 表格组件：允许自定义单元格内容
实际应用示例 - 数据表格组件
<div id="app">
  <data-table :data="users" :columns="columns">
    <template #name="{ value }">
      <strong>{{ value }}</strong>
    </template>

    <template #email="{ value }">
      <a :href="`mailto:${value}`">{{ value }}</a>
    </template>

    <template #actions="{ row }">
      <button @click="editUser(row.id)">编辑</button>
      <button @click="deleteUser(row.id)">删除</button>
    </template>
  </data-table>
</div>

<script>
const { createApp, ref } = Vue;

createApp({
    setup() {
      const users = ref([
        { id: 1, name: '张三', email: 'zhangsan@example.com', role: '管理员' },
        { id: 2, name: '李四', email: 'lisi@example.com', role: '用户' },
        { id: 3, name: '王五', email: 'wangwu@example.com', role: '编辑' }
      ]);

      const columns = ref([
        { key: 'name', title: '姓名', width: '150px' },
        { key: 'email', title: '邮箱', width: '200px' },
        { key: 'role', title: '角色', width: '100px' },
        { key: 'actions', title: '操作', width: '150px' }
      ]);

      const editUser = (userId) => {
        console.log('编辑用户:', userId);
      };

      const deleteUser = (userId) => {
        console.log('删除用户:', userId);
      };

      return {
        users,
        columns,
        editUser,
        deleteUser
      };
    }
    components: {
      DataTable: {
        props: {
          data: Array,
          columns: Array
        },
        template: `
          <table class="data-table">
            <thead>
              <tr>
                <th v-for="col in columns" :key="col.key" :style="{ width: col.width }">{{ col.title }}</th>
              </tr>
            </thead>
            <tbody>
              <tr v-for="row in data" :key="row.id">
                <td v-for="col in columns" :key="col.key">
                  <slot :name="col.key" :value="row[col.key]" :row="row">{{ row[col.key] }}</slot>
                </td>
              </tr>
            </tbody>
          </table>
        `
      }
    }
}).mount('#app');
</script>

最佳实践建议

在使用插槽时，有一些最佳实践建议：

  • 为插槽提供有意义的默认内容
  • 使用具名插槽时，给插槽起清晰的名称
  • 在作用域插槽中，只传递必要的数据
  • 合理使用v-slot指令，保持代码清晰
  • 对于复杂的组件，考虑使用多个插槽来增加灵活性
  • 文档化插槽的用途和可用的插槽Props

  • 创建一个表单组件，允许自定义表单字段
  • 创建一个列表组件，允许自定义列表项布局
  • 创建一个选项卡组件，允许自定义选项卡内容
