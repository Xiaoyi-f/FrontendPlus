组件基础

组件是Vue.js的核心概念之一，它让我们可以将复杂的UI拆分成独立的、可复用的部件。组件系统让我们能够用独立、解耦的方式构建大型应用，每个组件都可以看作一个小型的Vue实例。

组件基础核心概念

  • 组件定义：创建可复用的Vue组件
  • 组件注册：全局注册和局部注册组件
  • 组件复用：在应用中多次使用同一组件
  • 组件嵌套：在组件中使用其他组件
  • 组件模板：定义组件的结构和内容
  • 组件数据：组件的独立数据状态

组件的定义

组件本质上是一个拥有预定义选项的Vue实例。我们可以通过Vue.component()方法来全局注册一个组件。

基本组件定义示例
<div id="app">
  <button-counter></button-counter>
  <button-counter></button-counter>
  <button-counter></button-counter>
</div>

<script>
const { createApp } = Vue;

// 定义一个名为button-counter的组件
createApp({})
  .component('button-counter', {
    data() {
      return {
        count: 0
      }
    },
    template: `
      <div class="counter">
        <button @click="count++">点击次数: {{ count }}</button>
      </div>
    `
  })
  .mount('#app');
</script>

全局注册 vs 局部注册

组件可以通过两种方式注册：全局注册和局部注册。全局注册的组件可以在整个应用中的任何组件内部使用，而局部注册的组件只能在其注册的组件内部使用。

注册方式 优点 缺点 使用场景
全局注册 在任何组件中都可以使用 增加打包体积，可能造成命名冲突 通用组件，如按钮、输入框等
局部注册 按需引入，减少打包体积 需要在每个使用的地方导入 特定业务组件
全局注册示例
<script>
const { createApp } = Vue;

// 全局注册一个组件
createApp({})
  .component('my-button', {
    template: `
      <button class="btn">{{ buttonText }}</button>
    `,
    data() {
      return {
        buttonText: '点击我'
      }
    }
  })
  .mount('#app');
</script>
局部注册示例
<script>
const { createApp } = Vue;

// 定义组件
const MyButton = {
  template: `
    <button class="btn">{{ buttonText }}</button>
  `,
  data() {
    return {
      buttonText: '点击我'
    }
  }
};

// 局部注册组件
createApp({
  components: {
    MyButton // 等同于 'MyButton: MyButton'
  }
}).mount('#app');
</script>

组件的复用

组件最重要的特性之一就是可复用性。我们可以创建一个组件，然后在应用中多次使用它，每个实例都有自己的独立状态。

组件复用示例
<div id="app">
  <div class="component-list">
    <todo-item v-for="item in todoList" :key="item.id" :title="item.title"></todo-item>
  </div>
</div>

<script>
const { createApp, ref } = Vue;

createApp({
  setup() {
    const todoList = ref([
      { id: 1, title: '学习Vue组件' },
      { id: 2, title: '练习组件开发' },
      { id: 3, title: '构建组件库' }
    ]);

    return {
      todoList
    };
  }
  components: {
    TodoItem: {
      props: ['title'],
      template: `
        <div class="todo-item">
          <h3>{{ title }}</h3>
          <input type="checkbox" id="complete">
          <label for="complete">完成</label>
        </div>
      `
    }
  }
}).mount('#app');
</script>

组件嵌套

组件可以相互嵌套，形成组件树。这种嵌套关系有助于构建复杂的UI结构。

组件嵌套示例
<div id="app">
  <my-app></my-app>
</div>

<script>
const { createApp } = Vue;

// 定义子组件
const ChildComponent = {
  template: `
    <div class="child">
      <h3>子组件</h3>
      <p>这是嵌套的子组件内容</p>
    </div>
  `
};

// 定义父组件
const ParentComponent = {
  components: {
    ChildComponent
  },
  template: `
    <div class="parent">
      <h2>父组件</h2>
      <child-component></child-component>
    </div>
  `
};

// 定义根组件
createApp({
  components: {
    MyApp: ParentComponent
  }
}).mount('#app');
</script>

组件数据的独立性

每个组件实例都应该有自己独立的数据状态。在组件的data选项中，必须是一个返回数据对象的函数，而不是直接返回对象，这样可以确保每个组件实例都拥有独立的数据副本。

组件数据独立性示例
<div id="app">
  <counter-component></counter-component>
  <counter-component></counter-component>
  <counter-component></counter-component>
</div>

<script>
const { createApp } = Vue;

createApp({})
  .component('counter-component', {
    data() {
      // 注意：这里必须返回一个新的对象
      // 这样每个组件实例都有独立的数据副本
      return {
        count: 0
      }
    },
    template: `
      <div class="counter-component">
        <button @click="count++">点击: {{ count }}</button>
      </div>
    `
  })
  .mount('#app');
</script>

<style>
.counter-component {
  margin: 10px;
  padding: 10px;
  border: 1px solid #ccc;
  border-radius: 4px;
}
</style>

使用单文件组件(SFC)

在实际项目中，我们通常使用单文件组件(Single File Components，SFC)，它使用.vue文件扩展名，将模板、脚本和样式封装在一个文件中。

单文件组件示例
<!-- ButtonCounter.vue -->
<template>
  <div class="counter">
    <button @click="increment">点击次数: {{ count }}</button>
    <button @click="reset">重置</button>
  </div>
</template>

<script>
import { ref } from 'vue';

export default {
  name: 'ButtonCounter',
  setup() {
    const count = ref(0);

    const increment = () => {
      count.value++;
    };

    const reset = () => {
      count.value = 0;
    };

    return {
      count,
      increment,
      reset
    };
  }
};
</script>

<style scoped>
.counter {
  margin: 10px;
  padding: 10px;
  border: 1px solid #42b983;
  border-radius: 4px;
}

button {
  margin-right: 5px;
  padding: 5px 10px;
  background-color: #42b983;
  color: white;
  border: none;
  border-radius: 3px;
  cursor: pointer;
}
</style>

组合式API中的组件

在组合式API中，组件的定义方式略有不同，但核心概念保持一致。

选项式API
const MyComponent = {
  name: 'MyComponent',
  data() {
    return {
      count: 0
    }
  },
  methods: {
    increment() {
      this.count++;
    },
    reset() {
      this.count = 0;
    }
  }
};
组合式API
import { ref } from 'vue';

const MyComponent = {
  name: 'MyComponent',
  setup() {
    const count = ref(0);

    const increment = () => {
      count.value++;
    };

    const reset = () => {
      count.value = 0;
    };

    return {
      count,
      increment,
      reset
    };
  }
};

实际应用场景

组件在实际开发中有很多应用场景：

常见应用场景

  • UI组件库：如按钮、输入框、对话框等基础组件
  • 业务组件：如用户头像、商品卡片等特定业务组件
  • 布局组件：如导航栏、侧边栏、页脚等布局组件
  • 功能组件：如搜索框、购物车等具有特定功能的组件
  • 容器组件：用于组织和管理其他组件
实际应用示例 - 用户卡片组件
<div id="app">
  <user-card
    v-for="user in users"
    :key="user.id"
    :user="user"
    @user-click="handleUserClick">
  </user-card>
</div>

<script>
const { createApp, ref } = Vue;

createApp({
  setup() {
    const users = ref([
      { id: 1, name: '张三', email: 'zhangsan@example.com', avatar: 'https://via.placeholder.com/50' },
      { id: 2, name: '李四', email: 'lisi@example.com', avatar: 'https://via.placeholder.com/50' },
      { id: 3, name: '王五', email: 'wangwu@example.com', avatar: 'https://via.placeholder.com/50' }
    ]);

    const handleUserClick = (userId) => {
      alert(`点击了用户 ${userId}`);
    };

    return {
      users,
      handleUserClick
    };
  },
  components: {
    UserCard: {
      props: ['user'],
      template: `
        <div class="user-card" @click="$emit('user-click', user.id)">
          <img :src="user.avatar" :alt="user.name" class="avatar">
          <div class="user-info">
            <h3>{{ user.name }}</h3>
            <p>{{ user.email }}</p>
          </div>
        </div>
      `
    }
  }
}).mount('#app');
</script>

最佳实践建议

在使用组件时，有一些最佳实践建议：

  • 组件应该保持单一职责，只做一件事
  • 组件应该具有良好的可复用性
  • 组件名称应该具有描述性，使用PascalCase或kebab-case命名
  • 组件数据必须是函数，确保每个实例有独立的数据副本
  • 合理使用组件嵌套，避免过深的嵌套层级
  • 使用props进行父子组件通信，使用自定义事件向上传递数据

实践练习

  • 创建一个按钮组件，支持不同样式和大小
  • 创建一个卡片组件，可以显示不同类型的内容
  • 创建一个列表组件，支持不同的列表项
  • 创建一个表单组件，包含多个输入字段
  • 创建一个模态框组件，支持自定义内容
  • 创建一个导航组件，支持嵌套菜单
