组件通信

父子组件、兄弟组件通信方案

组件通信核心概念

  • Props：父组件向子组件传递数据
  • 自定义事件：子组件向父组件传递消息
  • Ref：父组件直接访问子组件
  • 事件总线：任意组件间的通信
  • Provide/Inject：跨层级组件通信
  • 状态管理：全局状态管理方案

组件通信概述

在Vue应用中，组件是构成UI的基本单元。由于组件是独立的，它们之间需要通信才能协同工作。Vue提供了多种组件通信方式，每种方式适用于不同的场景。

1. Props（父传子）

Props是父组件向子组件传递数据的主要方式。这是单向数据流的一部分，确保了组件的独立性。

// 子组件 const ChildComponent = { props: ['message', 'user'], // 声明接收的props template: ` <div> <p>{{ message }}</p> <p>用户: {{ user.name }} - {{ user.email }}</p> </div> ` } // 父组件 const ParentComponent = { template: ` <div> <child-component :message="greeting" :user="currentUser" ></child-component> </div> `, data() { return { greeting: 'Hello from parent!', currentUser: { name: 'Alice', email: 'alice@example.com' } } }, components: { 'child-component': ChildComponent } }

2. 自定义事件（子传父）

子组件通过$emit触发自定义事件，父组件监听这些事件来接收数据。

// 子组件 const ChildComponent = { data() { return { localValue: '' } }, template: ` <div> <input v-model="localValue" @input="onInput" /> </div> `, methods: { onInput() { // 触发自定义事件，向父组件传递数据 this.$emit('input-change', this.localValue) } } } // 父组件 const ParentComponent = { template: ` <div> <p>父组件接收到: {{ childData }}</p> <child-component @input-change="handleChildData"></child-component> </div> `, data() { return { childData: '' } }, methods: { handleChildData(value) { this.childData = value } }, components: { 'child-component': ChildComponent } }

3. Refs（父访问子）

通过ref特性，父组件可以直接访问子组件的属性和方法。

// 子组件 const ChildComponent = { data() { return { childValue: 'child data' } }, methods: { childMethod() { return 'Method called from child' } }, template: `<div>子组件内容</div>` } // 父组件 const ParentComponent = { template: ` <div> <child-component ref="childRef"></child-component> <button @click="accessChild">访问子组件</button> <p>子组件数据: {{ childData }}</p> </div> `, data() { return { childData: '' } }, methods: { accessChild() { // 通过$refs访问子组件的数据和方法 this.childData = this.$refs.childRef.childValue console.log(this.$refs.childRef.childMethod()) } }, components: { 'child-component': ChildComponent } }

4. 事件总线（Event Bus）

事件总线允许任意组件之间进行通信，但仅适用于简单场景，复杂应用应使用状态管理库。

// 创建事件总线 const EventBus = new Vue() // 组件A - 发送事件 const ComponentA = { template: `<button @click="sendData">发送数据</button>`, methods: { sendData() { EventBus.$emit('data-shared', { message: 'Hello from Component A' }) } } } // 组件B - 监听事件 const ComponentB = { template: `<div>接收到: {{ receivedMessage }}</div>`, data() { return { receivedMessage: '' } }, mounted() { // 监听事件 EventBus.$on('data-shared', (data) => { this.receivedMessage = data.message }) }, beforeDestroy() { // 移除事件监听器 EventBus.$off('data-shared') } } // 在Vue 3中，事件总线不再是内置功能，需要使用第三方库如mitt

5. Provide/Inject（跨层级通信）

Provide/Inject允许祖先组件向其所有子孙后代注入一个依赖，无论组件层级有多深。

// 祖先组件 const AncestorComponent = { provide() { return { theme: 'dark', user: this.user, // 提供方法 updateUser: this.updateUser } }, data() { return { user: { name: 'John', role: 'admin' } } }, methods: { updateUser(newUserData) { this.user = { ...this.user, ...newUserData } } }, template: `<div><slot></slot></div>` } // 后代组件 const DescendantComponent = { inject: ['theme', 'user', 'updateUser'], template: ` <div :class="theme"> <p>主题: {{ theme }}</p> <p>用户: {{ user.name }} - {{ user.role }}</p> <button @click="changeUser">更改用户</button> </div> `, methods: { changeUser() { this.updateUser({ name: 'Jane', role: 'user' }) } } }

6. Vuex/Pinia（全局状态管理）

对于复杂应用，全局状态管理是最有效的组件通信方式。

// 使用Pinia作为示例 import { defineStore } from 'pinia' // 定义store export const useMainStore = defineStore('main', { state: () => ({ user: { name: 'John', role: 'admin' }, theme: 'light' }), actions: { updateUser(userData) { this.user = { ...this.user, ...userData } }, toggleTheme() { this.theme = this.theme === 'light' ? 'dark' : 'light' } } }) // 任意组件中使用 const SomeComponent = { setup() { const store = useMainStore() return { user: computed(() => store.user), theme: computed(() => store.theme), updateUser: store.updateUser, toggleTheme: store.toggleTheme } } }

7. $parent/$children（不推荐）

虽然Vue提供了$parent和$children访问父/子组件，但这种方式耦合性强，不推荐使用。

// 不推荐的用法 const ChildComponent = { methods: { accessParent() { // 直接访问父组件数据（不推荐） this.$parent.parentData = 'changed from child' } } }

8. 通过共同父组件通信（兄弟组件）

兄弟组件可以通过共同的父组件进行通信。

// 兄弟组件A const SiblingA = { template: `<button @click="sendToSibling">发送给兄弟组件</button>`, methods: { sendToSibling() { // 通过$emit发送到父组件 this.$emit('data-from-a', 'Data from Sibling A') } } } // 兄弟组件B const SiblingB = { props: ['dataFromSibling'], template: `<div>来自兄弟组件的数据: {{ dataFromSibling }}</div>` } // 父组件 const Parent = { template: ` <div> <sibling-a @data-from-a="receiveDataFromA"></sibling-a> <sibling-b :data-from-sibling="sharedData"></sibling-b> </div> `, data() { return { sharedData: '' } }, methods: { receiveDataFromA(data) { this.sharedData = data } }, components: { 'sibling-a': SiblingA, 'sibling-b': SiblingB } }

Vue 3中的新通信方式

Vue 3引入了一些新的通信方式和改进：

// 使用Composition API进行通信 <script setup> import { provide, inject, ref } from 'vue' // 祖先组件 const theme = ref('light') provide('theme', theme) // 后代组件 const injectedTheme = inject('theme') </script> // 使用Teleport进行组件内容传送 <template> <teleport to="body"> <div class="modal">模态框内容</div> </teleport> </template>

通信方式选择指南

根据不同的场景选择合适的通信方式：

  • 父子组件通信：使用Props和自定义事件
  • 跨多层组件通信：使用Provide/Inject
  • 兄弟组件通信：通过共同父组件或状态管理
  • 任意组件通信：使用状态管理库（Vuex/Pinia）
  • 简单应用：可使用事件总线（Vue 2）或mitt库（Vue 3）
