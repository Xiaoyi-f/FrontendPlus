Suspense组件

Suspense是Vue 3.0引入的一个内置组件，它允许我们在等待异步组件加载时显示后备内容（fallback content）。Suspense让开发者可以优雅地处理异步组件加载状态，提供更好的用户体验。

Suspense核心概念

  • 异步组件加载：处理异步组件的加载状态
  • 后备内容：在加载期间显示的占位内容
  • 嵌套异步组件：等待多个嵌套异步组件加载
  • 错误处理：处理异步加载失败的情况
  • 组合式API支持：与async setup函数配合使用
  • 渐进式加载：逐步显示组件内容

Suspense工作原理

异步组件开始加载
→
显示fallback内容
→
异步组件加载完成
→
显示实际组件内容

Suspense组件通过捕获组件树中的异步依赖来工作。当Suspense内部的组件需要等待异步操作（如异步组件加载或组合式API中的异步setup函数）时，Suspense会显示后备内容，直到所有异步依赖都完成。

Suspense基本语法

插槽 说明 示例
default 异步内容，如果等待则会被挂起 <template #default>...</template>
fallback 在default插槽内容挂起时显示的内容 <template #fallback>...</template>
基本Suspense示例
<div id="app">
  <suspense-demo></suspense-demo>
</div>

<script>
const { createApp, defineAsyncComponent } = Vue;

// 模拟异步组件加载
const AsyncComponent = defineAsyncComponent(async () => {
  await new Promise(resolve => setTimeout(resolve, 2000));
  return {
    template: `
      <div class="async-content">
        <h4>异步加载的组件</h4>
        <p>内容已加载！</p>
      </div>
    `
  };
});

createApp({})
  .component('suspense-demo', {
    components: {
      AsyncComponent
    },
    template: `
      <div>
        <h3>Suspense基本示例</h3>
        <suspense>
          <template #default>
            <async-component></async-component>
          </template>
          <template #fallback>
            <div class="loading">加载中...</div>
          </template>
        </suspense>
      </div>
    `
  })
  .mount('#app');
</script>

Async Setup与Suspense

在组合式API中，我们可以使用async setup函数，这与Suspense配合使用非常有用。当setup函数是异步的，Suspense会等待其完成。

Async Setup与Suspense示例
<div id="app">
  <async-setup-demo></async-setup-demo>
</div>

<script>
const { createApp, ref } = Vue;

async function fetchUserData() {
  // 模拟API请求
  await new Promise(resolve => setTimeout(resolve, 1500));
  return {
    id: 1,
    name: 'Vue Developer',
    email: 'vue@example.com',
    role: 'Frontend Developer'
  };
}

createApp({})
  .component('async-setup-demo', {
    setup() {
      return async () => {
        const user = await fetchUserData();

        return {
          template: `
            <div class="user-info">
              <h4>{{ user.name }}</h4>
              <p>邮箱: {{ user.email }}</p>
              <p>角色: {{ user.role }}</p>
            </div>
          `,
          data() {
            return {
              user
            }
          }
        };
      }
    },
    template: `
      <div>
        <h3>Async Setup示例</h3>
        <suspense>
          <template #default>
            <async-setup-demo></async-setup-demo>
          </template>
          <template #fallback>
            <div class="loading">正在加载用户数据...</div>
          </template>
        </suspense>
      </div>
    `
  })
  .mount('#app');
</script>

Suspense事件

Suspense组件提供了resolving和resolve事件，可以在异步操作过程中执行相应逻辑。

Suspense事件处理示例
<div id="app">
  <suspense-event-demo></suspense-event-demo>
</div>

<script>
const { createApp, defineAsyncComponent } = Vue;

// 定义异步组件
const AsyncContent = defineAsyncComponent(async () => {
  await new Promise(resolve => setTimeout(resolve, 1500));
  return {
    template: `
      <div class="async-content">
        <h4>异步内容已加载</h4>
        <p>这个内容是异步加载的</p>
      </div>
    `
  };
});

createApp({})
  .component('suspense-event-demo', {
    data() {
      return {
        eventLog: []
      }
    },
    methods: {
      onResolve() {
        this.eventLog.push('组件已解析');
      },
      onResolving() {
        this.eventLog.push('正在解析组件');
      }
    },
    template: `
      <div>
        <h3>Suspense事件处理</h3>
        <suspense
          @resolve="onResolve"
          @resolving="onResolving">
          <template #default>
            <async-content></async-content>
          </template>
          <template #fallback>
            <div class="loading">加载中...</div>
          </template>
        </suspense>

        <div class="event-log">
          <h4>事件日志</h4>
          <ul>
            <li v-for="(event, index) in eventLog" :key="index">{{ event }}</li>
          </ul>
        </div>
      </div>
    `
  })
  .mount('#app');
</script>

嵌套Suspense

我们可以嵌套使用Suspense组件，以实现更精细的加载控制。这允许我们为不同层级的异步组件提供不同的加载状态。

嵌套Suspense示例
<div id="app">
  <nested-suspense-demo></nested-suspense-demo>
</div>

<script>
const { createApp, defineAsyncComponent } = Vue;

// 定义多层异步组件
const AsyncChild = defineAsyncComponent(async () => {
  await new Promise(resolve => setTimeout(resolve, 1000));
  return {
    template: `
      <div class="async-child">
        <h5>异步子组件</h5>
        <p>这是嵌套的异步组件</p>
      </div>
    `
  };
});

const AsyncParent = defineAsyncComponent(async () => {
  await new Promise(resolve => setTimeout(resolve, 1500));
  return {
    template: `
      <div class="async-parent">
        <h4>异步父组件</h4>
        <p>包含嵌套的异步子组件</p>
        <suspense>
          <template #default>
            <async-child></async-child>
          </template>
          <template #fallback>
            <div class="loading-child">正在加载子组件...</div>
          </template>
        </suspense>
      </div>
    `
  };
});

createApp({})
  .component('nested-suspense-demo', {
    components: {
      AsyncParent
    },
    template: `
      <div>
        <h3>嵌套Suspense示例</h3>
        <suspense>
          <template #default>
            <async-parent></async-parent>
          </template>
          <template #fallback>
            <div class="loading-parent">正在加载父组件...</div>
          </template>
        </suspense>
      </div>
    `
  })
  .mount('#app');
</script>

Suspense与错误处理

在实际应用中，异步加载可能会失败。虽然Suspense本身不直接处理错误，但我们可以通过组合其他技术来处理加载错误。

Suspense与错误处理示例
<div id="app">
  <error-handling-demo></error-handling-demo>
</div>

<script>
const { createApp, defineAsyncComponent } = Vue;

// 模拟可能失败的异步组件
const AsyncWithError = defineAsyncComponent(async () => {
  await new Promise(resolve => setTimeout(resolve, 1000));

  // 模拟50%的失败率
  if (Math.random() < 0.5) {
    throw new Error('异步加载失败');
  }

  return {
    template: `
      <div class="success-content">
        <h4>加载成功！</h4>
        <p>异步内容加载成功</p>
      </div>
    `
  };
});

createApp({})
  .component('error-handling-demo', {
    components: {
      AsyncWithError
    },
    data() {
      return {
        error: null,
        retryCount: 0
      }
    },
    methods: {
      handleError(error) {
        this.error = error.message;
        console.error('异步加载错误:', error);
      },
      retry() {
        this.error = null;
        this.retryCount++;
      }
    },
    template: `
      <div>
        <h3>错误处理示例</h3>
        <div class="error-section" v-if="error">
          <p class="error-message">错误: {{ error }}</p>
          <button @click="retry">重试</button>
        </div>
        <div v-else>
          <suspense>
            <template #default>
              <error-boundary @error="handleError">
                <async-with-error></async-with-error>
              </error-boundary>
            </template>
            <template #fallback>
              <div class="loading">加载中...</div>
            </template>
          </suspense>
        </div>
        <p>重试次数: {{ retryCount }}</p>
      </div>
    `
  })
  .component('error-boundary', {
    data() {
      return {
        error: null
      }
    },
    errorCaptured(err, vm, info) {
      this.error = err;
      this.$emit('error', err);
      return false; // 阻止错误继续冒泡
    },
    render() {
      return this.$slots.default;
    }
  })
  .mount('#app');
</script>

Suspense最佳实践

在使用Suspense时，有一些最佳实践可以帮助我们更好地利用其功能：

Suspense最佳实践

  • 提供有意义的加载状态：使用描述性的加载提示
  • 考虑用户体验：避免频繁的加载状态闪烁
  • 合理使用嵌套Suspense：为不同层级提供不同加载状态
  • 结合错误边界：处理异步加载失败的情况
  • 性能优化：考虑预加载和缓存策略
  • 渐进式加载：先加载主要内容，再加载次要内容

注意事项

使用Suspense时需要注意以下几点：

  • Suspense仅在Vue 3.0+版本中可用
  • 它只对异步组件、异步setup函数或异步数据获取有效
  • 在SSR期间，Suspense组件会渲染其默认插槽内容
  • 错误处理需要结合其他技术（如错误边界）来实现
  • 在某些复杂场景中，可能需要结合路由系统一起使用

实践练习

  • 创建一个异步数据加载的用户列表组件
  • 实现一个带有加载状态的图片懒加载组件
  • 创建一个异步加载内容的选项卡组件
  • 实现一个带有错误处理的异步表单提交
  • 创建一个分层加载的复杂页面结构
  • 实现一个带有进度指示的异步组件加载器
