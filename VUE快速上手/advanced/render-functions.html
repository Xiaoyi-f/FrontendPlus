渲染函数与JSX

渲染函数（Render Functions）和JSX是Vue.js提供的两种编程方式，用于替代模板语法。它们允许我们使用JavaScript的完整编程能力来创建组件，而不是使用基于字符串的模板。

渲染函数与JSX核心概念

  • 虚拟DOM：理解VNode（虚拟节点）的概念
  • 渲染函数：使用JavaScript函数创建VNode
  • JSX语法：在JavaScript中使用类似HTML的语法
  • createElement：创建虚拟DOM节点
  • 函数式组件：无状态组件的实现方式
  • 插槽渲染：以编程方式处理插槽内容

渲染函数工作原理

数据变化
→
执行render函数
→
生成VNode
→
对比旧VNode
→
更新真实DOM

渲染函数返回一个虚拟DOM节点（VNode），Vue会将这个VNode与上一次渲染的VNode进行比较，然后最小化更新真实DOM。渲染函数是模板编译后的最终形式，Vue的模板语法最终都会编译成渲染函数。

渲染函数基本语法

参数 类型 说明 示例
标签名 String HTML标签或组件名 'div', 'MyComponent'
数据对象 Object 包含DOM相关属性的配置 { class: 'my-class', attrs: { id: 'my-id' } }
子节点 Array|String 子节点数组或文本 [vnode1, vnode2] 或 'text'
基本渲染函数示例
<div id="app"></div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      count: 0
    }
  },

  methods: {
    increment() {
      this.count++;
    }
  },

  render(h) {
    // 使用渲染函数创建VNode
    return h(
      'div',
      { class: 'counter' },
      [
        h('h3', `计数: ${this.count}`),
        h('button', {
          on: { click: this.increment }
        }, '增加')
      ]
    );
  }
}).mount('#app');
</script>

JSX语法

JSX是一种JavaScript语法扩展，允许我们在JavaScript中编写类似HTML的代码。Vue 3支持使用JSX，但需要配置相应的编译器。

渲染函数语法
render() {
  return h(
    'div',
    { class: 'container' },
    [
      h('h2', '标题'),
      h('p', '内容')
    ]
  );
}
JSX语法
render() {
  return (
    <div class="container">
      <h2>标题</h2>
      <p>内容</p>
    </div>
  );
}
JSX语法示例
<div id="app"></div>

<script type="text/babel">
const { createApp } = Vue;

// 使用JSX语法的组件
const MyComponent = {
  data() {
    return {
      items: [
        { id: 1, text: '项目1' },
        { id: 2, text: '项目2' },
        { id: 3, text: '项目3' }
      ],
      title: '列表示例'
    }
  },

  methods: {
    addItem() {
      const newId = this.items.length + 1;
      this.items.push({
        id: newId,
        text: `项目${newId}`
      });
    }
  },

  render() {
    return (
      <div class="list-container">
        <h3>{this.title}</h3>
        <button onClick={ this.addItem }>添加项目</button>
        <ul>
          {this.items.map(item => (
            <li key={ item.id }>{ item.text }</li>
          ))}
        </ul>
      </div>
    );
  }
};

createApp(MyComponent).mount('#app');
</script>

渲染函数中的事件处理

在渲染函数中，事件处理器需要在数据对象的on属性中定义，这与模板中的@事件语法不同。

事件处理示例
<div id="app"></div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      message: 'Hello Vue!',
      count: 0
    }
  },

  methods: {
    updateMessage(event) {
      this.message = event.target.value;
    },

    increment() {
      this.count++;
    }
  },

  render(h) {
    return h('div', [
      h('h3', this.message),
      h('input', {
        attrs: {
          value: this.message,
          type: 'text'
        },
        on: {
          input: this.updateMessage
        }
      }),

      h('p', `计数: ${this.count}`),
      h('button', {
        on: { click: this.increment }
      }, '增加')
    ]);
  }
}).mount('#app');
</script>

插槽的渲染

在渲染函数中，插槽内容通过this.$slots和this.$scopedSlots访问，并且可以以编程方式渲染。

插槽渲染示例
<div id="app"></div>

<script>
const { createApp } = Vue;

// 插槽容器组件
const SlotContainer = {
  props: ['title'],
  render(h) {
    return h('div', { class: 'slot-container' }, [
      h('h3', this.$props.title),
      h('div', { class: 'default-slot' }, this.$slots.default),
      h('div', { class: 'footer-slot' }, this.$slots.footer)
    ]);
  }
};

// 作用域插槽组件
const ScopedSlotComponent = {
  data() {
    return {
      user: {
        name: 'Vue Developer',
        role: 'Frontend Developer'
      }
    }
  }
,   render(h) {
    // 渲染作用域插槽
    const scopedSlot = this.$scopedSlots.default;
    return scopedSlot ? scopedSlot(this.user) : h('div', '没有提供作用域插槽');
  }
};

createApp({
  components: {
    SlotContainer,
    ScopedSlotComponent
  },
  render(h) {
    return h('div', [
      h(SlotContainer, { props: { title: '插槽容器示例' } }, {
        default: [h('p', '这是默认插槽内容')],
        footer: [h('p', { class: 'footer' }, '这是页脚插槽内容')]
      }),

      h(ScopedSlotComponent, {
        scopedSlots: {
          default: (user) => [
            h('p', `姓名: ${user.name}`),
            h('p', `角色: ${user.role}`)
          ]
        }
      })
    ]);
  }
}).mount('#app');
</script>

函数式组件

函数式组件是没有自身状态的组件，它们接收输入并返回VNode。它们比普通组件更轻量，性能更好。

函数式组件示例
<div id="app"></div>

<script>
const { createApp } = Vue;

// 函数式组件
const FunctionalComponent = {
  functional: true,
  props: ['title', 'content'],
  render(h, { props, slots }) {
    return h('div', { class: 'functional-component' }, [
      h('h4', props.title),
      h('p', props.content),
      ...slots().default || []
    ]);
  }
};

createApp({
  components: {
    FunctionalComponent
  },
  render(h) {
    return h('div', [
      h(FunctionalComponent, {
        props: {
          title: '函数式组件示例',
          content: '这是一个函数式组件，没有自身状态'
        }
      }),
      h(FunctionalComponent, {
        props: {
          title: '另一个函数式组件',
          content: '性能更好，更轻量'
        }
      })
    ]);
  }
}).mount('#app');
</script>

渲染函数最佳实践

在使用渲染函数和JSX时，有一些最佳实践可以帮助我们更好地利用其功能：

渲染函数与JSX最佳实践

  • 选择合适的场景：用于复杂逻辑或动态组件
  • 保持简洁：避免在渲染函数中写过多逻辑
  • 合理使用JSX：在复杂模板中提高可读性
  • 性能考虑：使用key属性优化列表渲染
  • 类型安全：在TypeScript中使用类型定义
  • 函数式组件：在不需要状态时使用

何时使用渲染函数

渲染函数和JSX特别适用于以下场景：

  • 需要完全的编程控制能力，而不仅仅是模板语法
  • 创建动态组件，其结构基于复杂的逻辑
  • 封装可重用的渲染逻辑
  • 需要与不基于模板的库集成
  • 需要更灵活的组件抽象
