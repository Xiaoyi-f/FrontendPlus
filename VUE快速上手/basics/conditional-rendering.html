条件渲染

条件渲染是Vue.js的重要功能之一，它允许我们根据数据状态来决定是否渲染某些元素。Vue提供了v-if、v-else、v-else-if和v-show指令来实现条件渲染。

条件渲染核心概念

  • v-if：条件性地渲染元素，如果条件为假，则不会渲染元素
  • v-else：表示v-if的else分支，必须紧跟在v-if或v-else-if之后
  • v-else-if：表示v-if的else-if分支，必须紧跟在v-if或v-else-if之后
  • v-show：根据条件显示或隐藏元素，元素始终渲染并保留在DOM中
  • v-show vs v-if：v-if有更高的切换开销，v-show有更高的初始渲染开销
  • key属性：在条件渲染中使用key来强制替换元素

v-if指令

v-if指令用于条件性地渲染一块内容，这块内容只在指令的表达式返回真值时才被渲染。

v-if基本用法示例
<div id="app">
  <div v-if="isLoggedIn">欢迎回来，用户！</div>
  <div v-else>请先登录</div>

  <button @click="toggleLoginStatus">切换登录状态</button>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      isLoggedIn: false
    }
  },
  methods: {
    toggleLoginStatus() {
      this.isLoggedIn = !this.isLoggedIn;
    }
  }
}).mount('#app');
</script>

v-else-if指令

v-else-if提供的是v-if的else-if分支，可以链式使用多个。

v-else-if链式使用示例
<div id="app">
  <div v-if="type === 'A'">A</div>
  <div v-else-if="type === 'B'">B</div>
  <div v-else-if="type === 'C'">C</div>
  <div v-else>Not A/B/C</div>

  <button @click="setTypeA">设置为A</button>
  <button @click="setTypeB">设置为B</button>
  <button @click="setTypeC">设置为C</button>
  <button @click="setTypeD">设置为其他</button>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      type: 'A'
    }
  },
  methods: {
    setTypeA() { this.type = 'A'; },
    setTypeB() { this.type = 'B'; },
    setTypeC() { this.type = 'C'; },
    setTypeD() { this.type = 'D'; }
  }
}).mount('#app');
</script>

v-show指令

v-show也是根据条件展示元素，不同的是带有v-show的元素始终会被渲染并保留在DOM中，只是简单地切换CSS的display属性。

指令 描述 适用场景
v-if 条件性渲染元素，元素可能被创建或销毁 条件很少改变，需要完全移除/添加元素
v-show 切换元素的显示/隐藏，元素始终在DOM中 频繁切换显示/隐藏状态
v-show示例
<div id="app">
  <div v-show="isVisible">这个元素使用v-show控制</div>
  <div v-if="isVisible">这个元素使用v-if控制</div>

  <button @click="toggleVisibility">切换显示</button>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      isVisible: true
    }
  },
  methods: {
    toggleVisibility() {
      this.isVisible = !this.isVisible;
    }
  }
}).mount('#app');
</script>

使用template元素进行条件渲染

当需要条件性地渲染多个元素时，可以使用template元素作为包装器，最终的渲染结果不会包含template元素。

template条件渲染示例
<div id="app">
  <template v-if="user">
    <h1>{{ user.name }}</h1>
    <p>邮箱: {{ user.email }}</p>
    <p>年龄: {{ user.age }}</p>
  </template>

  <button @click="loadUser">加载用户信息</button>
  <button @click="clearUser">清除用户信息</button>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      user: null
    }
  },
  methods: {
    loadUser() {
      this.user = {
        name: '张三',
        email: 'zhangsan@example.com',
        age: 25
      };
    },
    clearUser() {
      this.user = null;
    }
  }
}).mount('#app');
</script>

key属性在条件渲染中的作用

在条件渲染中，Vue会尽可能高效地渲染元素，例如在切换v-if和v-else时，Vue会尽可能复用相同类型的元素。但有时我们希望阻止这种复用，这时可以使用key属性。

key属性示例
<div id="app">
  <div>
    <label>登录类型</label>
    <select v-model="loginType">
      <option value="username">用户名</option>
      <option value="email">邮箱</option>
    </select>
  </div>

  <!-- 没有key属性，Vue会复用input元素 -->
  <div v-if="loginType === 'username'">
    <label>用户名</label>
    <input placeholder="请输入用户名">
  </div>
  <div v-else>
    <label>邮箱</label>
    <input placeholder="请输入邮箱">
  </div>

  <!-- 有key属性，Vue会重新创建input元素 -->
  <div>
    <div v-if="loginType2 === 'username'">
      <label>用户名</label>
      <input key="username" placeholder="请输入用户名">
    </div>
    <div v-else>
      <label>邮箱</label>
      <input key="email" placeholder="请输入邮箱">
    </div>
  </div>

  <p>当前登录类型: {{ loginType }}</p>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      loginType: 'username',
      loginType2: 'username'
    }
  },
  watch: {
    loginType() {
      console.log('登录类型切换，但input值被保留');
    },
    loginType2() {
      console.log('登录类型2切换，input被重新创建');
    },
  }
}).mount('#app');
</script>

组合式API中的条件渲染

在组合式API中，条件渲染的语法是相同的，但数据定义方式不同。

选项式API
createApp({
  data() {
    return {
      showContent: true,
      userType: 'admin'
    }
  },
  methods: {
    toggleContent() {
      this.showContent = !this.showContent;
    }
  }
});
组合式API
createApp({
  setup() {
    const { ref } = Vue;

    const showContent = ref(true);
    const userType = ref('admin');

    const toggleContent = () => {
      showContent.value = !showContent.value;
    };

    return {
      showContent,
      userType,
      toggleContent
    };
  }
});

实际应用场景

条件渲染在实际开发中有很多应用场景：

常见应用场景

  • 用户认证：根据登录状态显示不同内容
  • 权限控制：根据用户权限显示不同功能
  • 表单验证：根据验证结果显示错误信息
  • 加载状态：根据数据加载状态显示不同UI
  • 响应式设计：根据屏幕尺寸显示不同组件
  • 错误处理：根据错误类型显示不同错误信息
实际应用示例 - 权限控制
<div id="app">
  <div class="user-info">
    <h2>用户信息</h2>
    <p>姓名: {{ user.name }}</p>
    <p>角色: {{ user.role }}</p>
  </div>

  <div v-if="user.role === 'admin'" class="admin-controls">
    <h3>管理员控制面板</h3>
    <button>管理用户</button>
    <button>系统设置</button>
    <button>数据备份</button>
  </div>

  <div v-else-if="user.role === 'editor'" class="editor-controls">
    <h3>编辑者控制面板</h3>
    <button>编辑内容</button>
    <button>审核文章</button>
  </div>

  <div v-else-if="user.role === 'user'" class="user-controls">
    <h3>用户控制面板</h3>
    <button>查看内容</button>
    <button>发表评论</button>
  </div>

  <div v-else class="guest-controls">
    <h3>访客控制面板</h3>
    <button>登录</button>
    <button>注册</button>
  </div>

  <div class="role-selector">
    <button @click="setRole('admin')">设为管理员</button>
    <button @click="setRole('editor')">设为编辑者</button>
    <button @click="setRole('user')">设为用户</button>
    <button @click="setRole('guest')">设为访客</button>
  </div>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      user: {
        name: '张三',
        role: 'user'
      }
    }
  },
  methods: {
    setRole(role) {
      this.user.role = role;
    }
  }
}).mount('#app');
</script>

性能优化建议

在使用条件渲染时，有一些性能优化的建议：

  • 对于频繁切换的元素，使用v-show比v-if更高效
  • 对于很少改变的元素，使用v-if可以减少初始渲染开销
  • 使用template元素包装多个条件渲染的元素
  • 正确使用key属性来控制元素的复用
  • 避免在条件渲染中使用复杂的计算表达式
