计算属性与侦听器

在Vue.js中，计算属性和侦听器是处理复杂逻辑和响应数据变化的重要工具。它们可以帮助我们更高效地处理数据依赖关系，提高应用性能。

计算属性与侦听器核心概念

  • 计算属性：基于响应式依赖进行缓存的属性，当依赖变化时自动更新
  • 侦听器：响应数据变化的回调函数，适合异步或开销较大的操作
  • 缓存机制：计算属性具有缓存特性，依赖未变化时不会重复计算
  • 性能优化：合理使用计算属性可以避免不必要的重复计算
  • 复杂逻辑处理：处理复杂的数据变换和依赖关系

计算属性 (Computed Properties)

计算属性是基于它们的响应式依赖进行缓存的。只有在相关响应式依赖发生改变时它们才会重新求值。

计算属性示例
<div id="app">
  <p>原始消息: {{ message }}</p>
  <p>反转消息: {{ reversedMessage }}</p>
  <p>反转消息(方法): {{ reverseMessage() }}</p>
  <button @click="updateMessage">更新消息</button>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      message: 'Hello Vue!'
    }
  },
  computed: {
    // 计算属性的getter
    reversedMessage() {
      console.log('计算反转消息...');
      return this.message.split('').reverse().join('');
    }
  },
  methods: {
    reverseMessage() {
      console.log('方法调用反转消息...');
      return this.message.split('').reverse().join('');
    },
    updateMessage() {
      this.message = 'Updated message ' + Date.now();
    }
  }
}).mount('#app');
</script>

计算属性 vs 方法

虽然计算属性和方法都能实现相同的功能，但计算属性会基于响应式依赖进行缓存，而方法每次都会执行。

计算属性的优势
  • 具有缓存机制，依赖未改变时直接返回缓存结果
  • 代码更简洁，使用方式与data中的属性相同
  • 自动追踪响应式依赖
  • 性能更好，避免重复计算
方法的使用场景
  • 需要接受参数的动态计算
  • 不需要缓存的计算
  • 需要在方法中执行复杂逻辑

计算属性的getter和setter

计算属性默认只有getter，但也可以定义setter，用于处理反向数据流。

计算属性的getter和setter
<div id="app">
  <p>{{ fullName }}</p>
  <button @click="updateFullName">更新全名</button>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      firstName: '张',
      lastName: '三'
    }
  },
  computed: {
    fullName: {
      // getter
      get() {
        return this.firstName + ' ' + this.lastName;
      },
      // setter
      set(newValue) {
        const names = newValue.split(' ');
        this.firstName = names[0];
        this.lastName = names[names.length - 1];
      }
    }
  },
  methods: {
    updateFullName() {
      // 这会触发setter
      this.fullName = '李 四';
    }
  }
}).mount('#app');
</script>

侦听器 (Watchers)

虽然计算属性在大多数情况下更合适，但有时我们需要在数据变化时执行异步或开销较大的操作，这时就需要使用侦听器。

侦听器示例
<div id="app">
  <p>问题: {{ question }}</p>
  <input v-model="question" placeholder="输入问题...">
  <p>答案: {{ answer }}</p>
</div>

<script src="https://unpkg.com/axios/dist/axios.min.js"></script>
<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      question: '',
      answer: '问题需要以问号结尾'
    }
  },
  watch: {
    // 侦听question的变化
    question(newQuestion, oldQuestion) {
      this.answer = '正在输入...';
      if (newQuestion.indexOf('?') !== -1) {
        this.getAnswer();
      }
    },
  },
  methods: {
    getAnswer() {
      // 模拟异步操作
      setTimeout(() => {
        this.answer = '这是模拟的答案';
      }, 1000);
    }
  }
}).mount('#app');
</script>

深度侦听

当需要侦听一个对象的深层属性变化时，可以使用deep选项。

深度侦听示例
<div id="app">
  <input v-model="user.name" placeholder="姓名">
  <input v-model="user.age" type="number" placeholder="年龄">
  <p>用户信息: {{ JSON.stringify(user) }}</p>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      user: {
        name: '张三',
        age: 25
      }
    }
  },
  watch: {
    user: {
      handler(newVal, oldVal) {
        console.log('用户信息变化:', newVal, oldVal);
      },
      deep: true, // 深度侦听
      immediate: true // 立即执行
    }
  }
}).mount('#app');
</script>

组合式API中的计算属性和侦听器

在组合式API中，我们可以使用computed函数和watch函数来实现相同的功能。

选项式API
createApp({
  data() {
    return {
      firstName: '张',
      lastName: '三'
    }
  },
  computed: {
    fullName() {
      return this.firstName + ' ' + this.lastName;
    }
  },
  watch: {
    firstName(newVal, oldVal) {
      console.log('firstName changed');
    }
  }
});
组合式API
createApp({
  setup() {
    const { ref, computed, watch } = Vue;

    const firstName = ref('张');
    const lastName = ref('三');

    // 计算属性
    const fullName = computed(() => {
      return firstName.value + ' ' + lastName.value;
    });

    // 侦听器
    watch(firstName, (newVal, oldVal) => {
      console.log('firstName changed');
    });

    return {
      firstName,
      lastName,
      fullName
    };
  }
});

计算属性的性能优势

计算属性的缓存机制使其在处理复杂逻辑时具有显著的性能优势。以下是一个示例来说明：

计算属性性能示例
<div id="app">
  <h3>原始列表: {{ list.length }} 项</h3>
  <div v-for="item in list" :key="item.id">{{ item.name }}</div>

  <h3>过滤后的列表 (计算属性): {{ filteredList.length }} 项</h3>
  <div v-for="item in filteredList" :key="item.id">{{ item.name }}</div>

  <h3>过滤后的列表 (方法): {{ getFilteredList().length }} 项</h3>
  <div v-for="item in getFilteredList()" :key="item.id">{{ item.name }}</div>

  <input v-model="filterText" placeholder="过滤文本">
  <button @click="addItems">添加项目</button>
</div>

<script>
const { createApp, ref, computed } = Vue;

createApp({
  setup() {
    const list = ref([]);
    const filterText = ref('');

    // 计算属性 - 有缓存
    const filteredList = computed(() => {
      console.log('计算过滤列表 (computed)');
      return list.value.filter(item =>
        item.name.toLowerCase().includes(filterText.value.toLowerCase())
      );
    });

    // 方法 - 每次都执行
    const getFilteredList = () => {
      console.log('执行过滤列表 (method)');
      return list.value.filter(item =>
        item.name.toLowerCase().includes(filterText.value.toLowerCase())
      );
    };

    const addItems = () => {
      for (let i = 0; i < 5; i++) {
        list.value.push({
          id: Date.now() + i,
          name: `项目 ${list.value.length + i + 1}`
        });
      }
    };

    // 初始化数据
    addItems();

    return {
      list,
      filterText,
      filteredList,
      getFilteredList,
      addItems
    };
  }
}).mount('#app');
</script>

侦听器的高级用法

Vue 3中的watch函数提供了更灵活的侦听能力。

侦听器高级用法示例
<script>
const { createApp, ref, watch, watchEffect } = Vue;

createApp({
  setup() {
    const x = ref(0);
    const y = ref(0);

    // 侦听单个源
    watch(x, (newX, oldX) => {
      console.log(`x changed from ${oldX} to ${newX}`);
    });

    // 侦听多个源
    watch([x, y], ([newX, newY], [oldX, oldY]) => {
      console.log(`x and y changed: [${oldX}, ${oldY}] -> [${newX}, ${newY}]`);
    });

    // watchEffect - 自动追踪依赖
    watchEffect(() => {
      console.log(`Current x: ${x.value}, y: ${y.value}`);
      // 会自动追踪x和y的变化
    });

    return { x, y };
  }
}).mount('#app');
</script>
