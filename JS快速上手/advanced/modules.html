模块化

模块化是现代JavaScript开发的核心概念，它允许我们将代码组织成可重用的模块，提高代码的可维护性、可读性和可重用性。ES6模块系统为JavaScript提供了标准的模块化方案。

模块化概览

  • ES6模块：JavaScript官方模块系统
  • CommonJS：Node.js使用的模块系统
  • 模块导出：命名导出和默认导出
  • 模块导入：按需导入模块功能
  • 模块化优势：代码组织、重用和维护

模块系统对比

模块系统 语法 运行环境 特点
ES6模块 import/export 现代浏览器/Node.js 静态分析，支持树摇
CommonJS require/module.exports Node.js 动态加载，运行时确定依赖
AMD define/require 浏览器 异步加载，适合浏览器环境

ES6模块系统

ES6模块是JavaScript官方的模块系统，使用import和export关键字来导入和导出模块。

// math.js - 导出模块

// 默认导出 - 每个模块只能有一个默认导出
export default function add(a, b) {
  return a + b;
}

// 命名导出 - 可以有多个命名导出
export const PI = 3.14159;

export function multiply(a, b) {
  return a * b;
}

export class Calculator {
  constructor() {
    this.history = [];
  }

  add(a, b) {
    const result = a + b;
    this.history.push({ operation: 'add', a, b, result });
    return result;
  }
}

// 还可以在文件末尾统一导出
const subtract = (a, b) => a - b;
const divide = (a, b) => {
  if (b === 0) throw new Error('除数不能为0');
  return a / b;
};

export { subtract, divide };
// main.js - 导入模块

// 导入默认导出
import add from './math.js';

// 导入命名导出
import { PI, multiply, Calculator, subtract, divide } from './math.js';

// 导入所有命名导出到一个对象
import * as mathUtils from './math.js';

// 混合导入
import add, { PI, multiply } from './math.js';

// 导入时重命名
import { multiply as mult, divide as div } from './math.js';

// 使用导入的功能
console.log(add(2, 3)); // 5
console.log(PI); // 3.14159
console.log(multiply(4, 5)); // 20

const calc = new Calculator();
console.log(calc.add(10, 5)); // 15

模块化高级用法

ES6模块系统提供了丰富的功能来满足复杂的模块化需求。

// 动态导入模块
async function loadCalculator() {
  try {
    const { Calculator } = await import('./math.js');
    const calc = new Calculator();
    return calc;
  } catch (error) {
    console.error('加载计算器模块失败:', error);
  }
}

// 条件导入
async function loadFeature(featureName) {
  let module;

  switch (featureName) {
    case 'chart':
      module = await import('./charts.js');
      break;
    case 'editor':
      module = await import('./editor.js');
      break;
    default:
      throw new Error(`未知功能: ${featureName}`);
  }

  return module;
}

// 在事件处理中使用动态导入
document.getElementById('load-chart-btn').addEventListener('click', async () => {
  const chartModule = await loadFeature('chart');
  chartModule.renderChart();
});
静态导入 vs 动态导入
  • 静态导入：编译时确定，提升性能，支持tree-shaking
  • 动态导入：运行时确定，按需加载，减少初始包大小
  • 语法：静态使用import声明，动态使用import()函数
  • 时机：静态在文件顶部，动态可在任何地方
  • 条件性：静态无法条件导入，动态可条件导入
模块系统对比
  • ES6模块：标准，静态分析友好，浏览器原生支持
  • CommonJS：Node.js，运行时求值，支持动态导入
  • AMD：浏览器，异步加载，现已较少使用
  • UMD：通用模块定义，兼容多种模块系统

工具函数模块设计
// utils/string.js - 字符串处理工具

export function capitalize(str) {
  if (typeof str !== 'string') return str;
  return str.charAt(0).toUpperCase() + str.slice(1).toLowerCase();
}

export function truncate(str, maxLength, suffix = '...') {
  if (str.length <= maxLength) return str;
  return str.substring(0, maxLength - suffix.length) + suffix;
}

export function camelCase(str) {
  return str
    .replace(/[^a-zA-Z0-9]+/g, ' ')
    .trim()
    .split(' ')
    .map((word, index) => {
      if (index === 0) return word.toLowerCase();
      return word.charAt(0).toUpperCase() + word.slice(1).toLowerCase();
    })
    .join('');
}

// 默认导出一个包含所有函数的对象
export default {
  capitalize,
  truncate,
  camelCase
};
API客户端模块设计
// api/client.js - API客户端模块

class ApiClient {
  constructor(baseUrl, options = {}) {
    this.baseUrl = baseUrl;
    this.defaultHeaders = options.headers || {};
    this.timeout = options.timeout || 5000;
  }

  async request(endpoint, options = {}) {
    const url = `${this.baseUrl}${endpoint}`;
    const config = {
      headers: {
        'Content-Type': 'application/json',
        ...this.defaultHeaders,
        ...options.headers,
      },
      ...options,
    };

    if (options.body && typeof options.body === 'object') {
      config.body = JSON.stringify(options.body);
    }

    try {
      const controller = new AbortController();
      const timeoutId = setTimeout(() => controller.abort(), this.timeout);

      config.signal = controller.signal;

      const response = await fetch(url, config);
      clearTimeout(timeoutId);

      if (!response.ok) {
        throw new Error(`HTTP错误! 状态: ${response.status}`);
      }

      return await response.json();
    } catch (error) {
      if (error.name === 'AbortError') {
        throw new Error('请求超时');
      }
      throw error;
    }
  }

  get(endpoint) {
    return this.request(endpoint, { method: 'GET' });
  }

  post(endpoint, data) {
    return this.request(endpoint, {
      method: 'POST',
      body: data
    });
  }

  put(endpoint, data) {
    return this.request(endpoint, {
      method: 'PUT',
      body: data
    });
  }

  delete(endpoint) {
    return this.request(endpoint, { method: 'DELETE' });
  }
}

// 默认导出
export default ApiClient;

// 预配置的API客户端
export const createApiClient = (baseUrl) => new ApiClient(baseUrl);

// 通用API客户端
export const defaultClient = new ApiClient('https://api.example.com');

模块打包和构建

在实际项目中，我们通常使用构建工具来处理模块，如Webpack、Rollup或Vite。

// package.json - 项目配置

{
  "name": "my-app",
  "version": "1.0.0",
  "type": "module", // 启用ES6模块
  "main": "dist/index.js",
  "scripts": {
    "build": "webpack --mode production",
    "dev": "webpack serve --mode development"
  }
}

// webpack.config.js - Webpack配置

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      }
    ]
  },
  mode: 'development'
};

模块化最佳实践

在使用模块化时，有一些最佳实践需要注意：

  • 保持模块单一职责，每个模块只做一件事
  • 使用有意义的模块名称
  • 合理组织模块依赖关系，避免循环依赖
  • 导出的内容应该是稳定的API
  • 考虑使用tree-shaking来减少包大小
  • 为模块编写适当的文档
